# Splits Buyback & Burn

Connect any existing splits contract to automated buyback & burn — without giving up custody.

EarnKit provisions a Turnkey-managed wallet and sets it as the fee recipient on your splits contract. From that point on, every fee distribution that lands in the wallet is automatically swapped to your target token via Uniswap V3 and burned — once the configurable USD threshold is crossed.

<Callout type="info">
  **Platform fee: 5% per execution.** The fee is deducted from the total swap input before the buyback runs. For example, if $100 has accumulated, $5 goes to the platform and $95 is swapped into your token and burned.
</Callout>

---

## Supported Contracts

| Protocol | Contract type | Notes |
|----------|--------------|-------|
| Clanker | Push-split (0xSplits V2) | Fee recipient must be set to EarnKit wallet |
| Doppler | Pull-split (0xSplits V2) | Standard warehouse withdraw supported |
| Custom 0xSplits V2 | Push or pull | Any V2 split with a configurable fee recipient |

## How It Works

```
Fees distributed to split contract
  └── EarnKit buyback wallet receives its allocation
           └── threshold crossed?
                ├── yes → deduct 5% platform fee
                │         → withdraw from Splits Warehouse (if pull-split)
                │         → swap ETH / USDC / MINIDEV → your token (Uniswap V3)
                │         → burn (burn() or transfer to 0x000...dEaD)
                └── no  → accumulates until threshold
```

### Step by step

1. **Register** — paste your splits contract address in the EarnKit dashboard. A Turnkey-managed buyback wallet is provisioned for you.

2. **Set fee recipient** — update the fee recipient on your splits contract to point to the provisioned wallet address. EarnKit shows you exactly what transaction to send (or pre-fills a Safe TX Builder link if you're using a multisig).

3. **Fees accumulate** — every distribution that reaches the wallet is tracked. EarnKit checks the swappable USD value after each receipt.

4. **Threshold triggered** — once the wallet's swappable balance crosses your configured threshold (minimum $10, default $10), a buyback execution is queued.

5. **Swap & burn** — the platform fee is deducted, assets are withdrawn from the Splits Warehouse (for pull-splits), swapped to your target token via the best available Uniswap V3 route, and burned.

## Setting Up

You configure buybacks at registration time in the [EarnKit dashboard](https://app.earnkit.com).

| Setting | Description |
|---------|-------------|
| **Splits contract address** | The 0xSplits V2 contract already receiving your fees |
| **Target token address** | The ERC-20 token to buy and burn on Base |
| **USD threshold** | Minimum swappable balance before execution triggers (min $10) |
| **Recovery address** | Where to sweep funds if execution fails after all retries |

<Callout type="warning">
  The buyback wallet address is embedded at registration time and cannot be changed. If you need to update it, you would need to re-register and update the fee recipient on your contract again.
</Callout>

## Execution States

| State | What's happening |
|-------|-----------------|
| `PENDING` | Registered — waiting for the first fee receipt |
| `ACTIVE` | Monitoring — accumulating fees, threshold not yet crossed |
| `QUEUED` | Threshold crossed — execution scheduled |
| `WITHDRAWING` | Pulling assets from Splits Warehouse to the buyback wallet |
| `SWAPPING` | Swapping ETH / USDC / MINIDEV → your token via Uniswap V3 |
| `BURNING` | Calling `burn()` or transferring to `0x000...dEaD` |
| `COMPLETED` | Tokens burned successfully |
| `RETRYING` | A step failed — retrying with backoff (up to 5 attempts: 1m, 5m, 30m, 2h, 12h) |
| `FAILED` | All retries exhausted — use "Recover stranded funds" to retrieve assets |
| `BROKEN` | Permanent error detected — contact support or re-register |

## Supported Input Tokens

The buyback wallet accepts ETH, USDC, and MINIDEV as fee inputs. Each is independently swapped to your buyback token. If one asset's swap fails, the others still proceed.

**Special case:** if fees arrive already denominated in your target token, no swap is needed — the executor proceeds directly to burn.

## Security Design

- **Non-custodial** — you remain the owner of your splits contract. The buyback wallet is only the fee recipient, not a contract admin.
- **Immutable wallet** — the buyback wallet address cannot be changed post-registration. No server compromise can redirect your fees.
- **Operator gas drip** — EarnKit drips a small ETH reserve (0.0001 ETH) before each execution. Gas cost is excluded from the swappable balance check so it never falsely triggers the threshold.
- **Per-asset swap isolation** — each token is swapped in its own try/catch. A failed USDC route will not block ETH or MINIDEV from being swapped and burned.

## FAQ

**Can I change the target token after registering?**
No. The buyback wallet and its Turnkey policy are fixed at registration. You would need to re-register.

**What happens if my token doesn't have a `burn()` function?**
EarnKit transfers the tokens to `0x000000000000000000000000000000000000dEaD` (the dead address).

**What if execution fails permanently?**
After 5 retries, use **Recover stranded funds** in the dashboard to sweep all assets to your recovery address.

**Can I use any ERC-20 as my target token?**
Yes — any ERC-20 on Base with a Uniswap V3 pool. The executor finds the best route automatically (direct or two-hop via WETH).

**I use Clanker and all view functions on my split revert. Will this work?**
Yes. Clanker uses push-splits where state lives in the factory, not on the contract. EarnKit handles this — executions are activated manually via the dashboard after your fee recipient is set.
